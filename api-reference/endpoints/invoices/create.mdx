title: 'Create Invoice'

## Request

<ParamField body="client_id" type="string" placeholder="1a2b3c4d5e">
  The ID of an existing client. Either `client_id` or `client_data` is required.
</ParamField>

<ParamField body="client_data" type="object">
  Create a new client inline with the invoice. Either `client_id` or `client_data` is required.
  
  <Expandable title="Client data properties">
    <ParamField body="type" type="string" required>
      Client type. Options: `b2b`, `b2c`, `business`, `individual`, `person`
    </ParamField>
    
    <ParamField body="name" type="string" required>
      Client name or company name
    </ParamField>
    
    <ParamField body="cui" type="string">
      Tax ID (CUI/CIF) - required for b2b clients
    </ParamField>
    
    <ParamField body="rc" type="string">
      Registration code (e.g., J40/1234/2023)
    </ParamField>
    
    <ParamField body="address" type="string" required>
      Street address
    </ParamField>
    
    <ParamField body="city" type="string">
      City name
    </ParamField>
    
    <ParamField body="county" type="string">
      County or state
    </ParamField>
    
    <ParamField body="country" type="string" default="RO">
      Country code (ISO 3166-1 alpha-2)
    </ParamField>
    
    <ParamField body="postal_code" type="string">
      Postal or ZIP code
    </ParamField>
    
    <ParamField body="email" type="string">
      Email address
    </ParamField>
    
    <ParamField body="phone" type="string">
      Phone number
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="series_id" type="string">
  Invoice series CzUid. If not specified, the default series for the document type will be used.
</ParamField>

<ParamField body="items" type="array" required>
  Array of invoice line items
  
  <Expandable title="Item properties">
    <ParamField body="description" type="string" required>
      Product or service description
    </ParamField>
    
    <ParamField body="quantity" type="number" required>
      Quantity
    </ParamField>
    
    <ParamField body="price" type="number" required>
      Unit price (without VAT)
    </ParamField>
    
    <ParamField body="vat_rate" type="number" default="21">
      VAT percentage. Available rates: 21% (default), 19%, 11%, 9%, 5%, 0%
    </ParamField>
    
    <ParamField body="unit" type="string" default="buc">
      Unit of measure (e.g., buc, ore, kg)
    </ParamField>
    
    <ParamField body="save_as_product" type="boolean" default="false">
      Save this item as a product for future use
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="document_type" type="string" default="fiscal">
  Document type. Options: `fiscal`, `proforma`
</ParamField>

<ParamField body="date" type="string" default="today">
  Invoice date in YYYY-MM-DD format
</ParamField>

<ParamField body="due_date" type="string" default="30 days">
  Due date in YYYY-MM-DD format
</ParamField>

<ParamField body="currency" type="string" default="RON">
  Currency code (ISO 4217)
</ParamField>

<ParamField body="observations" type="string">
  Additional notes or payment instructions
</ParamField>

<ParamField body="is_draft" type="boolean" default="false">
  Save as draft without finalizing
</ParamField>

<ParamField body="efactura_enabled" type="boolean" default="true">
  Enable e-Factura submission to ANAF
</ParamField>

<ParamField body="spv_environment" type="string" default="live">
  SPV environment: `live` (production) or `test` (SPV testing). Determines which OAuth keys to use for e-Factura submission.
</ParamField>

<Warning>
  E-Factura requires OAuth authentication. Configure it at `/firm/index/efactura` in your Contazen account. Without OAuth, e-Factura submissions will fail with an `oauth_not_configured` error.
</Warning>

<ParamField body="send_email" type="boolean" default="false">
  Send invoice by email after creation
</ParamField>

<ParamField body="email_address" type="string">
  Override recipient email (uses client email by default)
</ParamField>

## VAT Rates

### Available VAT Rates in Romania

| Rate | Type | Description | Available From |
|------|------|-------------|----------------|
| 21% | Standard | Default rate | August 1, 2025 |
| 19% | Standard | Current standard rate | Current |
| 11% | Reduced | Food services, accommodation | August 1, 2025 |
| 9% | Reduced | Medicines, books, etc. | Current |
| 5% | Reduced | Social housing | Current |
| 0% | Exempt | Various exemptions | Current |

<Note>
  The 11% and 21% VAT rates will be available starting August 1, 2025. Until then, use 19% as the standard rate.
</Note>

### VAT Key Mapping

VAT keys are automatically determined based on the VAT rate, but can be manually specified:

- Key 1: 19% Standard
- Key 2: 9% Reduced
- Key 3: 5% Reduced
- Key 4: 0% Exempt
- Key 5: 0% SFDD
- Key 6: 0% SDD
- Key 7: 0% VAT Included
- Key 8: 0% Reverse Charge (taxare inversă)
- Key 9: 21% Standard (from August 2025)
- Key 11: 11% Reduced (from August 2025)
